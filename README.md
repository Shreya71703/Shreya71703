## ◈ Featured Projects

<details>
<summary><b>◆ AI Learning Assistant</b> — Production RAG-powered Study Platform</summary>

<br/>

An enterprise-grade AI learning platform built as a full-stack application with a React frontend, FastAPI backend, and a complete RAG pipeline. Users upload academic PDFs, which are processed via OCR, chunked semantically, and embedded into a persistent ChromaDB vector store. The system supports intelligent Q&A, AI-generated quizzes, flashcards, summaries, and personalised study plans — all served through a containerized, cloud-deployed architecture.

| Attribute | Detail |
|-----------|--------|
| **Stack** | React · Vite · FastAPI · ChromaDB · Sentence Transformers · Docker · Hugging Face Spaces |
| **Scale** | Multi-document ingestion · Persistent vector store · JWT-authenticated multi-user |
| **Performance** | Model pre-baked in Docker image · Sub-second semantic retrieval · Async API design |
| **Security** | JWT Bearer token auth · bcrypt password hashing · CORS-hardened API |
| **Impact** | Deployed to Hugging Face Spaces · Full production CI via GitHub to HF Space pipeline |
| **Repository** | [Shreya71703/AI-Learning-Assistant](https://github.com/Shreya71703/AI-Learning-Assistant) |

The platform architecture follows a clean separation of concerns: the React SPA is compiled and served as static assets by the FastAPI backend in production, eliminating the need for a separate frontend server. The RAG pipeline chains document chunking, embedding, ChromaDB upsert, cosine-similarity retrieval, and LLM-augmented response generation. The Dockerfile uses a multi-stage build to keep the final image lean, with the sentence transformer model baked in at build time to guarantee zero-latency cold starts on constrained cloud hardware.
